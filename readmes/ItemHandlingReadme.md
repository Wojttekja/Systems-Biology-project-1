# Item Handling Instruction

## Short Explanation

This is Instruction for what should be mentioned in **Every** Pull Request during **Any** Item Handling.

---

## Mid Explanation

### Every Pull Request Should be Reviewed by Other Contributor.

This means that when @Wojttekja creates Pull Request, i should review it, and vice-versa

### Every Task in Action Plan Should be worked out on Designated Branch.

This means that if @Karolcha100 wants to work with #8 He needs to create New Branch (Under The Item).

### What Pull Request should Contain?

1. Information about item, under what it was developed.
2. Information about what has been done.
3. Future Work (which can be marked with [TODO or NoNeed])

---

## Long Explanation

Following **Mid Explanation** correct form of pull request should look like that:

```
## Item under which it was developed`
- [Item Reference]

---

## What was done?

1. [Description]
2. [Description]
...

## Future Work

[Future Work Description or "[TODO or NoNeed]"]

```
